Use Cases

Manage Accounts

Use Case: Manage Member

Context: The software at the ChocAn Data Center is run allowing operators to add new members to ChocAn, to delete members who have resigned, and to update member records.

Actors: Operator

Main Success Scenario:
 1. An operator is prompted to add, delete, or update member
 2. The operator enters information of the member
 3. The operator adds, deletes, or updates the member

Extensions:
 2a. The member does not exist in the system
     An error message is displayed on the screen then goes back to step 1
 2b. The member is suspended
     An error message id displayed and goes back to step 1
 2c. When adding a member, the member’s information already exists in the system
     The software alerts the operator then allow access to the provider to go to step 3

Use Case: Manage Provider

Context: Similar to the use case as above, the software at the ChocAn Data Center is run allowing operators to now add new providers, to delete providers, and to update provider records.

Actors: Operator

Main Success Scenario:
 1. An operator is promoted to add, delete, or update provider
 2. The operator enters information of the provider
 3. The operator adds, deletes, or updates the provider

Extensions:
 2a. The provider does not exist in the system
     An error message is displayed on the screen then goes back to step 1
 2b. When adding a provider, the provider’s information already exists in the system
     The software alerts the operator then allows access to the provider to go to step 3

Verification

Use Case: Verify Service Code

Context: When the provider enters in the six-digit service code, the provider interacts with the system to verify the service code.

Actors: Provider

Main Success Scenario:
 1. The system finds the service name that corresponds to the service code.
 2. The system displays the service name to the terminal.
 3. The system prompts the provider to verify that the service name matches with the service that was provided.
 4. The provider confirms that the service name and service provided match.
     4.1 The provider enters comments about the service provided.

Extensions:
 1a. A nonexistent service code was entered.
     1a.1 An error message is printed to the terminal.
     1a.2 The use case terminates.
 3a. The service name does not match with the service that was provided
     3a.1. The use case terminates.

Use Case: Complete Service Form

Context: The provider completes a form with information related to the service provided.

Actors: Provider

Main Success Scenario:
 1. The terminal displays a form that includes the current date and time, the date the service was provided, member name and number, service code, and fee to be paid to the terminal.
 2. The provider submits the completed form.

Healthcare Services

Provider Directory Lookup

Use Case: Request Provider Directory

Context: A provider can request a Provider Directory, a list of services, at any time.

Actors: Provider

Main Success Scenario:
 1. The provider requests the software product for a Provider Directory.
     1.1 The Provider Directory alphabetically lists the service names and corresponding codes and fees.
 2. The Provider Directory is sent as an e-mail attachment to the provider.

Use Case: Access Provider Directory

Context: The provider can access the Provider Directory to look up service information and to bill ChocAn.

Actors: Provider, Member

Main Success Scenario:
 1. The provider swipes the Member's card to access the provided services.
 2. The provider enters in the date the service was provided.
 3. The provider then accesses the Provider Directory to look up the corresponding 6 digit service code.
 4. The service code is keyed in and verified by the provider.
     4.1 The provider enters comments about the service provided.

Extensions:
 4a. The provider entered the wrong service code and verification was unsuccessful.
     4a.1. Repeat step 4.

Accounting

Glossary

Member : A user who receives service from ChocAn.

Member Report : A file for each member with name, number, address, and the service information for that week.

Manager : Handles Provider Report's and the Summary Report.

Member Record : A record of all ChocAn members in which operators can add new members, delete resigned members, and update member information.

Summary Report : a report given to the manager for accounts payable. Lists every provider to be paid that week, the number of consultations each had, and his or her total fee for that week

Provider : a health care professional who provides services to members

Provider Directory : a file created to provide an ordered list of service names and corresponding service codes and fees

Provider Record : A record of all ChocAn service providers in which operators can add, delete, or update provider information.

Provider Report : A weekly file for each provider containing that provider's personal information and consultation and service description for each service provided that week. The service description contains the total number of consultations provided and the total fee for the week.

Electronic Funds Transfer Record : A disk containing each provider's bank information and the amount to be paid.

Operator : Handles the Member and Provider Record's to add, delete, or update information.
